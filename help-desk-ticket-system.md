# 🎫 Help Desk Ticket System Simulation

## 📌 Project Overview
This project simulates a real-world IT Help Desk environment where users submit tickets and IT support resolves them. It demonstrates my ability to troubleshoot technical issues, document solutions, and follow a structured support process.

---

## 🎯 Objective
To practice handling common IT support issues while following a help desk workflow, including ticket creation, troubleshooting, and resolution.

---

## 🛠️ Tools & Skills Used
- Ticketing system concepts (ServiceNow, Jira)
- Troubleshooting methodology
- Customer support communication
- Documentation and ticket notes
- Basic networking and system diagnostics

---

## 🧾 Scenario 1: Password Reset Request

### Issue
A user reports they are unable to log into their account after multiple failed attempts.

### Steps Taken
1. Verified user identity
2. Checked account lockout status
3. Reset user password
4. Instructed user to create a strong password
5. Confirmed successful login

### Resolution
User regained access to their account after password reset.

---

## 🧾 Scenario 2: Slow Computer Performance

### Issue
User reports their computer is running very slowly.

### Steps Taken
1. Checked running applications
2. Identified high CPU usage
3. Closed unnecessary programs
4. Performed system restart
5. Recommended disk cleanup

### Resolution
System performance improved after closing background applications and restarting.

---

## 🧾 Scenario 3: Network Connectivity Issue

### Issue
User is unable to connect to the internet.

### Steps Taken
1. Checked Wi-Fi connection status
2. Restarted router
3. Ran network diagnostics
4. Renewed IP configuration
5. Tested connectivity

### Resolution
Internet connection restored after resetting network configuration.
