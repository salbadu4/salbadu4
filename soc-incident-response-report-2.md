# SOC Incident Response Report – Suspicious File Download

## 📌 Overview
This fun practice project demonstrates my ability to analyze a security alert and respond to a potential threat using Security Operations Center (SOC) principles. The objective was to investigate a SIEM alert, determine its validity, and follow incident response procedures to contain and remediate the issue while ahowing the ability to document everything thoroughly.
---

## 🚨 Scenario
A SIEM alert was triggered indicating a suspicious file download on a company workstation. The alert flagged a domain associated with known malicious activity.

As a SOC Analyst, I was responsible for investigating the alert, determining its validity, and taking appropriate response actions to protect the organization.

---

## 🔔 Alert Details
- **Alert Type:** Suspicious File Download  
- **Severity:** Medium to High  
- **Source:** SIEM System  
- **Affected Asset:** Employee Workstation  
- **Date/Time:** February 2026  

---

## 🔍 Investigation Steps
- Reviewed SIEM alert details including timestamps and affected systems  
- Analyzed user activity related to the download event  
- Checked the domain reputation using threat intelligence concepts  
- Identified potential indicators of compromise (IOCs)  
- Determined whether the activity was malicious or a false positive  

---

## 🚨 Findings
The domain involved in the download was identified as potentially malicious. The user downloaded a file from an untrusted external source, which could pose a risk to the system.

There was no immediate evidence of system compromise, but the activity was considered suspicious and required further monitoring and response.

---

## 🛠 Response Actions
- Isolated the affected workstation from the network to prevent further risk  
- Removed the suspicious file from the system  
- Initiated a malware scan on the affected device  
- Reset user credentials as a precautionary measure  
- Monitored system activity for any further suspicious behavior  

---

## 📊 Recommendations
- Educate users on safe browsing and download practices  
- Implement web filtering to block known malicious domains  
- Enhance SIEM alert monitoring and response procedures  
- Enforce endpoint protection and antivirus policies  
- Enable multi-factor authentication (MFA) for user accounts  

---

## ✅ Conclusion
The incident was successfully investigated and contained with no confirmed compromise. Preventative measures were identified to reduce the likelihood of similar incidents in the future.
